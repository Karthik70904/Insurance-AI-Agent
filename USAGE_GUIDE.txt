HOW IT WORKS
------------
1. User types a message
2. AI agent analyzes the query
3. Detects claim numbers, policy numbers, or keywords
4. Retrieves data from Supabase database
5. Formats and displays the response
6. Saves conversation history

TECHNICAL DETAILS
-----------------
Frontend:    Pure HTML5, CSS3, JavaScript (ES6+)
Backend:     Supabase (PostgreSQL database)
API:         Supabase REST API
Storage:     Browser sessionStorage for session management
Database:    5 tables (policies, claims, faqs, conversations, escalations)

CUSTOMIZATION
-------------

TROUBLESHOOTING
---------------
Issue: Chat not responding
Fix:   Check browser console for errors, verify Supabase config

Issue: Claim/Policy not found
Fix:   Ensure correct format (CLM-2024-XXXX or POL-2024-XXX)

Issue: Styling issues
Fix:   Clear browser cache and reload

SECURITY NOTES
--------------
- Supabase credentials in config.js are public (anon key)
- Row Level Security (RLS) enabled on all tables
- For production: Use environment variables, enable HTTPS
- No sensitive data should be stored in browser
